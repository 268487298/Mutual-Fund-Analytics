# AGENTS.md

Purpose
-------
This file gives concise, actionable guidance for AI coding agents working on this repository. It favours linking to repository docs over embedding long explanations and describes expectations when asked to act (especially when the argument `gemini` is provided).

Quick setup
-----------
- **Activate venv (Windows PowerShell):** `mutual\Scripts\Activate.ps1` (or use your preferred virtualenv).  
- **Install deps:** `pip install -r requirements.txt`.
- **Run ETL:** `python main.py` (validates CSVs and loads `bluestock_mf.db`).

Key files & locations
---------------------
- **Entry point:** [main.py](main.py) — loads core CSVs into `bluestock_mf.db`.  
- **Data (raw):** `data/raw/` — contains source CSVs (do not modify without approval).  
- **Processed:** `data/processed/` — derived CSVs generated by notebooks.  
- **Scripts:** `scripts/` — helpers (see `scripts/live_nav_fetch.py`).  
- **Notebooks:** `notebooks/` — exploratory and analysis artifacts.  
- **Schema:** [sql/schema.sql](sql/schema.sql) — DB schema reference.  
- **Requirements:** [requirements.txt](requirements.txt)

Agent behavior (general)
-----------------------
- **Read first:** Always read `README.md`, this `AGENTS.md`, and `main.py` before proposing changes.  
- **Link, don't embed:** Link to existing docs; do not copy long docs into PRs.  
- **State-changing actions:** Ask the user for confirmation before running commands that modify data or the DB (e.g., `python main.py`, writing to `data/`, or committing large files).  
- **Patches preferred:** When proposing code changes produce an `apply_patch`-style diff or small focused edits, not a full-file rewrite without explanation.  
- **Small iterative steps:** Prefer incremental PRs with tests (if added) rather than large, risky changes.
- **No raw data exfiltration:** Do not upload or externalize raw data files; summarize statistics instead.

Agent behavior (for the `gemini` argument)
-----------------------------------------
When the user supplies the argument `gemini`, the agent should additionally:
- Use concise, numbered steps for actions and keep replies short.  
- Always surface environment activation and dependency commands before any run.  
- Prefer offering to create an `AGENTS.md` or skill file (this file) and ask for explicit approval to run stateful commands.  
- If generating analysis or model outputs, include reproducible commands and indicate expected runtime and resource use.
 - **Gemini instructions:** [agents/gemini.agent.md](agents/gemini.agent.md) — agent checklist, examples, and quick-run commands.

Repository conventions & pitfalls
-------------------------------
- The repo recreates `bluestock_mf.db` on `main.py` runs — avoid accidental overwrites.  
- Notebooks are primary analysis artifacts; convert critical notebook logic to scripts before automated runs.  
- Large CSVs live under `data/raw/`; consider adding them to `.gitignore` if they grow.
